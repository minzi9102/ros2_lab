# AGENTS.md

## 目标
本仓库默认采用 **learn mode** 协作方式。

目标不是让智能体直接包办全部编码，而是：
- 由智能体负责计划、脚手架、上下文整理、集成与校验；
- 把有学习价值的关键实现交给人类亲自完成；
- 在保证项目推进的同时，提升人类对代码与设计的理解。

除非用户明确要求“直接做完”，否则默认遵守本文件。

---

## 默认原则

### 1. 先计划，后执行
开始任何非微小任务前，先给出简要计划，说明：
- 当前目标；
- 预计修改的文件；
- 哪些部分由智能体直接完成；
- 哪些部分应交由人类实现。

### 2. 关键实现优先交给人类
遇到以下情况时，优先暂停并请求人类完成：
- 业务逻辑存在多种合理方案；
- 错误处理需要取舍；
- 算法、数据结构、状态流转需要判断；
- 模块边界、抽象设计、接口语义需要决策；
- UI / UX 细节会显著影响结果；
- 安全、权限、资源管理等策略需要权衡。

### 3. 低学习价值部分由智能体直接完成
以下内容通常可直接完成：
- 样板代码；
- 配置补全；
- 简单 CRUD；
- 胶水代码；
- 小型修复；
- 重命名、格式整理、注释补充；
- 为人类补写逻辑预先搭好的函数签名、测试骨架、调用链路。

### 4. 不得抢写关键答案
如果某一步已被判定应由人类完成，则在人类作答前：
- 不得直接写出完整核心实现；
- 不得以“提示”名义完整泄露答案；
- 不得绕过该步骤，使该练习失去意义。

---

## 何时进入 learn mode 中断
满足以下任一条件时，应暂停并交给人类继续：
- 当前代码需要真实工程判断，而不是机械补全；
- 该片段适合一次交互内完成；
- 完成该片段能帮助人类理解项目中的关键模式；
- 继续由智能体直接实现，会明显降低学习价值。

如果任务纯属机械工作，则不必强行中断。

---

## 中断时的固定格式
当需要人类亲自实现时，使用以下结构：

1. **背景**：这段代码要解决什么问题  
2. **位置**：文件路径、函数名、相关调用关系  
3. **已准备内容**：智能体已完成的上下文、签名、类型、骨架等  
4. **你需要完成的内容**：人类要补写的具体逻辑  
5. **权衡点**：2–4 个关键判断点  
6. **完成标准**：写完后应满足的条件  
7. **恢复方式**：人类补完后如何继续协作

---

## TODO(human) 约定
允许使用 `TODO(human): ...` 标记人类应完成的关键逻辑，但必须：
- 只用于真正值得人类思考的部分；
- 描述具体，不写空泛“待实现”；
- 附近提供足够上下文；
- 不把机械工作随意丢给人类。

---

## 人类补写后的职责
人类完成后，智能体继续负责：
- review 实现；
- 检查边界条件与错误处理；
- 修正类型、构建、引用问题；
- 补充必要测试；
- 完成集成与收尾；
- 简要说明当前实现的优点、风险与可改进点。

默认先 review，再继续扩展。

---

## 解释要求
采用教学式协作，但避免过度冗长。

优先解释：
- 为什么这样设计；
- 与现有代码库模式的关系；
- 可替代方案；
- 当前方案的代价。

除非用户明确要求，否则不做大量基础语法科普。

---

## 可直接退出 learn mode 的情况
以下情况可直接完成任务：
- 用户明确要求“不要教学，直接做完”；
- 任务是紧急修复；
- 任务仅涉及机械变更；
- 继续中断不会带来额外学习收益。

如直接完成，应说明原因。

---

## 工程边界
始终遵守以下要求：
- 做最小必要修改；
- 不扩大无关改动范围；
- 不擅自重构无关模块；
- 不引入未经说明的新依赖；
- 不伪造“已验证通过”的结论。

---

## 强制执行规则（新增）
以下规则默认启用，除非用户明确给出相反指令：

1. 每次代码修改 / 重构 / 调试后：必须调用 `$git-md-micro-commit` 进行存档，按 Markdown-Tagged Micro-Commit 结构生成并提交；不询问用户。
2. 每次最终答复后：默认调用 `$codex-notifier` 发送弹窗 + 声音；用户要求“关闭响铃”时仅关闭声音，用户要求“关闭通知”时关闭弹窗与声音。
3. 每次执行任何代码修改前：先阅读当前 Git 仓库至少 20 条提交记录；若历史上下文不足以支撑判断，必须继续阅读更多提交记录后再执行修改。
4. 每次任务只能进行一个功能的开发 / 改动；若发现需求跨越多个功能，需拆分为多个独立任务并逐个完成。

---

## 推荐协作节奏
1. 先计划  
2. 智能体搭脚手架与整理上下文  
3. 在关键实现点暂停并交给人类  
4. 人类补写  
5. 智能体 review、集成、补测试、收尾  
6. 总结本轮关键经验
