# 任务 4A 规划文档：Service 最小闭环

## 1. 任务目标
- 完成一个最小可运行的 ROS 2 Service Server + Client 闭环。
- 明确 Topic / Service / Action 三种通信方式在机械臂控制场景中的边界。

## 2. 任务范围
- 包含：服务接口定义、服务端状态维护、客户端调用、基础异常返回。
- 不包含：真实机器人控制、复杂状态机、多线程优化。

## 3. 前置条件
- 已完成 `colcon build` 与工作区环境加载。
- 熟悉当前学习包目录结构与基本命令。

## 4. 实施步骤
1. 新建练习包并定义服务语义（如 `set_mode`）。
2. 实现 Service Server：接收请求并回写执行结果。
3. 实现 Service Client：发送请求并输出响应。
4. 编写最小 launch，一次启动 server/client（或分步启动）。
5. 记录行为差异：该场景为什么优先选 Service。
6. 将过程与结论同步写入实验文档。

## 5. 交付物
- 代码：最小 Service 包与可运行节点。
- 文档：`notes/labs/task4A_service_vs_topic_action.md`。

## 6. 验收标准
- 能稳定发起请求并收到可解释响应。
- 能清楚说明为何“轨迹执行不适合用 Service”。

## 7. 风险与回退
- 风险：接口语义设计过重，导致实现复杂。
- 回退：退回单一命令语义，仅保留最小字段验证闭环。
