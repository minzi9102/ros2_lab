# 任务 4B 规划文档：QoS 对比实验

## 1. 任务目标
- 通过可重复实验理解 QoS 对通信行为的影响。
- 能根据数据流类型选择合理 QoS，而不是沿用默认值。

## 2. 任务范围
- 包含：`reliable` / `best_effort`、`history` / `depth` 的最小对比。
- 不包含：DDS 厂商级深入调优、跨网络复杂压测。

## 3. 前置条件
- 已具备基础 Topic 发布/订阅经验。
- 可运行至少一组发布者与订阅者节点。

## 4. 实施步骤
1. 设计最小实验矩阵（至少 2x2）。
2. 搭建发布与订阅节点，参数化 QoS。
3. 在相同负载下采集日志与现象。
4. 记录丢包、延迟、稳定性差异。
5. 总结“传感数据 vs 控制指令”推荐 QoS。

## 5. 交付物
- 代码：QoS 对比用最小节点或参数化脚本。
- 文档：`notes/labs/task4B_qos_experiment.md`。

## 6. 验收标准
- 能给出至少两类场景的 QoS 选择依据。
- 文档中有可复现实验步骤与结果记录。

## 7. 风险与回退
- 风险：实验变量过多导致结论不清晰。
- 回退：固定变量，只比较一组关键参数。
