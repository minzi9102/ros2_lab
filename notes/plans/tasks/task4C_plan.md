# 任务 4C 规划文档：tf2 主动查询

## 1. 任务目标
- 实现 `base_link -> tool0` 的主动 TF 查询。
- 掌握 tf2 常见异常处理（超时、外推、坐标系不存在）。

## 2. 任务范围
- 包含：TF Buffer/Listener、定时查询、错误处理与日志。
- 不包含：复杂标定链、多机器人命名空间冲突处理。

## 3. 前置条件
- 已有 robot_state_publisher 与 joint_states 运行链路。
- 已能在 RViz 中观察 TF 树。

## 4. 实施步骤
1. 新建最小 tf2 查询节点（Python 或 C++）。
2. 周期查询目标变换并打印位姿。
3. 为查询增加超时与异常分支。
4. 对照 RViz 结果验证查询一致性。
5. 记录时间戳与同步注意事项。

## 5. 交付物
- 代码：tf2 查询最小节点。
- 文档：`notes/labs/task4C_tf2_lookup.md`。

## 6. 验收标准
- 查询成功时输出稳定位姿。
- 异常时日志可直接定位问题类型。

## 7. 风险与回退
- 风险：话题 remap 或 frame 命名不一致。
- 回退：先在默认 `/tf` 上验证，再恢复隔离话题。
