# 任务 4F 规划文档：joint_trajectory_controller 联调

## 1. 任务目标
- 将现有 FollowJointTrajectory Client 接入本地控制器执行链路。
- 完成一次完整的 goal -> feedback -> result 闭环验证。

## 2. 任务范围
- 包含：action name 对齐、关节名校验、轨迹点时间参数校验。
- 不包含：复杂运动学约束优化、碰撞规避规划。

## 3. 前置条件
- 4E 已完成，控制器已可用。
- Action Client 基础功能可运行。

## 4. 实施步骤
1. 校对控制器 action 接口名与关节顺序。
2. 准备 2-3 个平滑轨迹点进行联调。
3. 发送目标并观察 feedback/result。
4. 记录失败场景并形成排障步骤。
5. 回写验收结论与改进项。

## 5. 交付物
- 代码：可执行的最小轨迹联调链路。
- 文档：`notes/runbooks/follow_joint_trajectory_debug.md`。

## 6. 验收标准
- 目标可被接受并返回结果。
- 常见失败能通过 runbook 快速定位。

## 7. 风险与回退
- 风险：关节名与控制器配置不一致。
- 回退：先用最小单点目标验证接口连通，再恢复多点轨迹。
