# 任务 4G 规划文档：forward_command_controller 对比

## 1. 任务目标
- 对比 `joint_trajectory_controller` 与 `forward_command_controller` 的行为差异。
- 输出在不同开发阶段的控制器选择建议。

## 2. 任务范围
- 包含：相同场景下的控制方式对比与记录。
- 不包含：多控制器并发切换的复杂编排策略。

## 3. 前置条件
- 已完成 4F，具备轨迹控制联调基础。
- 能独立查看控制器状态与关键话题。

## 4. 实施步骤
1. 设定统一测试动作与对比维度。
2. 分别运行两类控制器并记录现象。
3. 对比精度、可控性、实现复杂度。
4. 总结各自适用场景与风险边界。
5. 更新任务文档与结论。

## 5. 交付物
- 文档：`notes/labs/task4G_controller_comparison.md`。

## 6. 验收标准
- 能明确回答“什么场景选哪种控制器”。
- 有可复现的对比步骤与结论依据。

## 7. 风险与回退
- 风险：测试条件不一致导致结论偏差。
- 回退：固定同一输入轨迹与同一采样窗口重复实验。
